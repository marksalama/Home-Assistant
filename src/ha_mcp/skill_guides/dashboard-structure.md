# Dashboard structure

Keep dashboards practical and scannable.

- Start with one overview view and one view per common workflow or area.
- Use entity cards for direct status/control, history graphs for trends, and
  markdown only for short operational notes.
- Keep destructive or heavy actions grouped separately.
- Prefer stable entity IDs and labels/areas so automations and dashboards stay
  readable.
- After saving dashboards, verify the dashboard exists with `list_dashboards`
  and `get_dashboard`.
